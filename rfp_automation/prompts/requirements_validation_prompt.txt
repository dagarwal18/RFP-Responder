2. AMBIGUITY — Flag requirements that use imprecise language (e.g.,

3. DUPLICATES — Identify requirements that say the same thing in

4. CONTRADICTIONS — Identify requirements that conflict with each other.

5. CONFIDENCE SCORE — Provide an overall confidence score (0.0 to 1.0)
   representing how reliable and well-formed the requirement set is.

   - 0.5-0.7 = several vague or duplicate entries
   - Below 0.5 = significant quality problems

   the ORIGINAL RFP TEXT above. If a requirement's value does NOT match
