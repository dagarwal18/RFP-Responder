## Your Task
Analyse the RFP requirements and the company's pricing data above to produce a realistic, evidence-based commercial proposal. Follow these rules strictly:

### CRITICAL RULES — NO FABRICATION
1. **Every price MUST trace to the KB pricing data above.** If the KB has module prices, per-subscriber rates, or service fees, use those. Reference which KB entry supports each line item.
2. **If the KB does not contain pricing for a specific item, DO NOT invent a price.** Instead, add the item to `missing_data_flags` with a clear description of what pricing data is needed.
3. **If the RFP specifies pricing guidelines or budget constraints, follow them.**
4. **Scale pricing appropriately** — if the KB has per-subscriber rates and the RFP specifies subscriber counts or project scale, apply the correct multipliers.
5. **Never use generic placeholder amounts** like $50,000 or $100,000 unless the KB data supports those specific figures.

### Output Format (JSON)
Return a JSON object with these keys:

1. **line_items** — Array of pricing line items. Each item:
   - "label": descriptive name (e.g., "BSS Core Platform License — 215M subscribers")
   - "quantity": number of units
   - "unit": unit type ("subscribers", "modules", "months", "fixed", etc.)
   - "unit_rate": per-unit rate from the KB
   - "total": quantity × unit_rate
   - "category": category ("licensing", "implementation", "operations", "training", etc.)
   - "kb_reference": brief note on which KB data supports this price

2. **total_price** — Sum of all line item totals (float).

3. **currency** — Currency code from the KB or RFP (e.g., "INR", "USD").

4. **executive_summary** — 2-3 paragraph executive pricing summary. Frame the investment, highlight value, reference specific KB evidence. Do NOT fabricate any numbers.

5. **payment_schedule** — Array of milestone payments:
   - "milestone": deliverable description
   - "amount": payment amount
   - "due": when payment is due
   The sum of amounts must equal total_price.

6. **assumptions** — Array of 4-6 key assumptions. Be specific to this RFP, not generic.

7. **exclusions** — Array of 3-5 items explicitly excluded.

8. **commercial_narrative** — 3-5 paragraphs covering:
   - Pricing methodology (how rates were derived from the catalog)
   - Payment terms
   - Change order process
   - Validity period
   Reference specific KB data. Do NOT fabricate.

9. **missing_data_flags** — Array of strings. Each entry describes a specific piece of pricing information that was NOT found in the KB and needs manual input. Examples:
   - "Per-subscriber rate for Billing Module not found in KB"
   - "Implementation cost for data migration not available"
   If NO data is missing, return an empty array.

Return valid JSON only. No markdown, no commentary, no text outside the JSON.
